# 记账系统优化记录

今天继续优化我的记账系统：

```text
闫的记账铺
```

这次不是单纯加功能，而是把真正使用时不舒服的地方重新整理了一遍。  
主要优化了四个地方：同步逻辑、出行统计、账单页面比例、首页信息结构。

![记账系统优化后的首页](/image/jizhang-opt-28.webp)

## 同步逻辑改成合并模式

之前电脑和手机都能同步，但有一个隐患：  
如果某一台设备先点了“上传本机到云端”，有可能把另一台设备已经上传过的数据覆盖掉。

这次把同步方式改成了更稳的合并逻辑：

```text
上传本机到云端 = 把本机账单合并到云端
从云端同步到本机 = 把云端账单合并回本机
```

这样电脑和手机都可以记账，不会因为点错按钮就把另一端的数据冲掉。

![电脑端同步设置](/image/jizhang-opt-29.webp)

![手机端同步设置](/image/jizhang-opt-30.webp)

## 出行统计兼容单人和双人

这次还补了出行记账场景。

如果只是自己出行，就关闭双人统计，系统会默认把出行账单算到“本人”，界面也不会显示同行人相关内容。

如果是两个人一起出行，就打开：

```text
开启双人出行统计
```

出行账单就可以选择：

```text
本人
同行人
```

这样统计页就能分开看两个人各自花了多少钱。

![双人出行统计开关](/image/jizhang-opt-31.webp)

## 账单页面比例修好了

今天还修了一个很明显的问题：  
账单页面在手机端和电脑端都会出现比例不协调，文字被挤到一边，甚至变成竖排。

后来发现是复选框被全局样式撑得太大，影响了整条账单的布局。

现在重新调整后：

- 复选框固定大小
- 分类图标不乱跑
- 金额不会挤压备注
- 操作按钮不会把内容推歪
- 手机端和电脑端都更正常

![账单页面优化后效果](/image/jizhang-opt-32.webp)

## 首页重新精简

首页之前堆了太多信息。  
比如净资产、预算、账户数量、分类数量都放在首页，会让页面变长，而且手机端看起来很挤。

这次首页改成更适合日常打开的结构：

```text
今日收支结余
今日收入
今日支出
本月收入
本月支出
消费类型占比
近期账单
```

我觉得这样更符合记账习惯：  
每天打开先看今天花了多少、收入多少，再看本月整体情况。

![首页日/月收支概览](/image/jizhang-opt-33.webp)
