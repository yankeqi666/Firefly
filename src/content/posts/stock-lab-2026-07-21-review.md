# Stock Lab 专业量化复盘工作台开发记录

这篇文章记录的是我最近做的一个股票复盘工具：**Stock Lab 专业量化复盘工作台**。

一开始它只是一个很简单的“小白股票翻译机”想法：输入股票代码，把看不懂的行情和指标翻译成大白话。后来越做越发现，如果只是翻译几个指标，其实意义不大。真正有用的地方应该是：

- 今天市场到底是偏强、偏弱，还是震荡；
- 我持仓里哪只股票风险最大；
- 哪只股票仓位过重；
- 是否跌破风险观察线；
- 是继续观察、减仓降风险，还是可以加仓观察；
- 分析依据来自哪里，哪些数据还缺失。

所以这个项目最后逐步改成了一个更接近移动端操盘软件的 **AI 股票复盘工作台**。

## 一、项目最终定位

这个系统不是荐股软件，也不是自动交易软件。

它的定位是：

```text
帮普通投资者把行情、持仓、风险线、市场环境和仓位纪律整理清楚。
```

我希望它像一个“收盘后的复盘助手”：

- 不替用户下单；
- 不保证涨跌；
- 但是敢给出明确判断；
- 判断必须带依据；
- 判断必须有反证条件；
- 数据取不到时不能编造。

也就是说，它不是一句“投资有风险”就结束，而是要真的告诉用户：

```text
现在为什么偏强？
哪里说明它可能错了？
如果继续持有，应该看哪条线？
如果仓位过重，应该先处理什么？
```

## 二、现在已经实现的功能

### 1. 市场温度和牛熊判断

系统会读取几个主要指数：

- 上证指数
- 深证成指
- 创业板指
- 科创50

然后结合当日涨跌、上涨指数数量、下跌指数数量、成交额，判断当前市场环境：

```text
牛市环境 / 熊市环境 / 震荡市
```

如果市场环境好，界面会用偏红色的提示；如果环境不好，会用偏蓝色的提示；如果市场方向不清楚，则使用灰色或中性提示。

> 图片说明：这里放市场温度模块截图，重点展示牛市/熊市/震荡市、当日成交额和指数涨跌。

### 2. 个股专业复盘结论

单只股票分析不只是显示 MA、RSI、涨跌幅，而是先给出一个明确判断标签，例如：

- 牛市偏多
- 熊市防守
- 偏强观察
- 高位复核
- 偏弱防守
- 震荡等待

然后系统会列出判断依据：

- 当前价格
- MA10 / MA20 / 60日压力
- RSI
- 量能
- 主力资金流
- 市场环境

最重要的是，它会给出反证条件。例如：

```text
如果跌破 MA20 且放量，原有观察逻辑降级。
如果冲到 60 日压力附近但量能跟不上，说明上方抛压仍重。
如果重新站回 MA20，偏弱判断可以降级为修复观察。
```

![个股专业复盘结论截图，占位图](/image/42.webp)

> 图片说明：这里放个股分析页截图，重点展示专业复盘结论、风险观察线和加减仓倾向。

### 3. 持仓风险预警机

用户可以手动录入自己的持仓：

- 股票代码
- 成本价
- 股数
- 账户总额

系统会根据这些信息计算：

- 当前市值
- 浮盈亏
- 仓位占比
- 是否重仓
- 是否跌破 MA20 风险观察线
- 是否接近 60 日压力位

然后给出持仓建议，例如：

- 可加仓观察
- 继续持有偏多
- 继续持有观察
- 减仓降风险
- 压力位减仓
- 防守回避

这些建议会结合真实市场环境。比如熊市环境下，即使个股反弹，也会更偏向防守；牛市环境下，如果趋势和资金都配合，才会提高观察积极度。

![持仓预警机截图，占位图](/image/43.webp)

### 4. 仓位管理和板块建议

后面我又补了一个仓位管理模块，因为只看单只股票还不够。

系统现在会计算：

- 总仓位
- 最大单只股票仓位
- 板块集中度
- 哪个板块占比最高

板块识别会优先用行情接口返回的行业字段，如果取不到，就根据名称做简单识别，例如：

- 白酒消费
- 银行金融
- 券商金融
- 半导体
- 新能源
- 医药
- ETF

这样可以发现一种很常见的问题：

```text
不是单只股票看起来有风险，而是整个账户过度集中在一个板块。
```

### 5. 截图 OCR 识别持仓

为了降低使用门槛，我还加了截图识别持仓。

用户可以上传券商持仓截图，系统会在浏览器本地用 OCR 识别：

- 股票代码
- 股票名称
- 成本价
- 股数

这个功能不会把图片上传到服务器，识别是在用户浏览器本地完成的。不过 OCR 不可能百分百准确，所以识别后仍然需要人工核对。

## 三、技术实现方式

这个项目目前是纯前端 + Cloudflare Pages Functions 的轻量方案。

主要文件包括：

```text
index.html
styles.css
script.js
functions/api/analyze.js
README.md
```

部署方式是：

```text
GitHub 仓库 + Cloudflare Pages
```

这样做的好处是：

- 不需要买服务器；
- 不需要数据库；
- 不需要域名备案；
- 使用 Cloudflare 自带的 pages.dev 域名即可访问；
- 每个人打开网页即可使用；
- 持仓数据保存在用户自己的浏览器本地。

当前访问地址是：

```text
https://stock-lab-bd0.pages.dev/?v=22
```

## 四、数据来源和真实性处理

这个系统尽量使用公开行情接口读取真实数据，包括：

- 实时行情
- 历史 K 线
- 指数行情
- 当日成交额
- 主力资金流字段
- 新闻标题
- 公告/财报线索
- 财务摘要

但是我也给系统加了“数据可靠性”判断。

如果某些数据没有取到，系统不会假装自己知道，而是会提示：

```text
数据不足
需复核
置信度降低
```

我觉得这一点很重要。因为金融分析最怕两件事：

1. 没有数据却装作有数据；
2. 有一点数据就说得特别绝对。

所以这个系统的原则是：

```text
可以判断，但要说明依据。
可以犯错，但不能编造。
可以给建议，但要有反证条件。
```

## 五、为什么做了很多版

这个项目不是一版做成的。

大概经历了二十多次迭代，从最早的“小白翻译机”，一直改到现在的 `v22`。

中间主要经历了这些变化：

### v1 - v6：从想法到基础页面

最早只是想做一个简单网页，输入股票代码，然后显示一些基础行情。

这个阶段的问题是：

- 页面太普通；
- 分析太浅；
- 只是展示数据，没有真正帮人判断。

### v7 - v12：加入持仓和复盘

后来加入了持仓功能，可以保存股票成本价和股数，也可以生成简单复盘。

这个阶段开始有一点实用性，但仍然不够：

- 建议太保守；
- 分析像模板；
- 没有市场环境；
- 没有仓位管理。

### v13 - v18：开始做专业判断

这一阶段加入了：

- 风险观察线
- MA20
- 60 日压力位
- 持仓建议
- 专业复盘结论
- 数据置信度

系统开始敢给出判断，例如偏强观察、偏弱防守、高位复核等。

### v19 - v20：重做界面风格

这个阶段主要改界面。

我希望它更像 iOS 上的股票或券商 App，而不是一个普通网页后台。

所以调整了：

- 深色操盘台首屏；
- iOS 风格底部导航；
- 更紧凑的数字卡片；
- 专业复盘结论重点卡；
- 市场温度深色面板。

### v21：加入真实市场环境

这一版很关键。

系统开始根据真实指数和成交额判断：

```text
牛市环境 / 熊市环境 / 震荡市
```

并且根据市场环境调整持仓建议。

### v22：加入仓位管理、板块建议和截图识别

这是目前最新版。

加入了：

- 总仓位；
- 最大单只仓位；
- 板块集中度；
- 板块建议；
- 截图 OCR 识别持仓。

做到这里以后，这个工具才比较像一个“每天可以打开看一眼”的股票复盘工具。

## 六、开发中最难的地方

这个项目难的地方不只是写代码。

更难的是不断判断：

```text
到底要做成一个玩具，还是做成一个真的能辅助复盘的工具？
```

过程中有几个地方反复改了很多次：

### 1. 分析不能太空

一开始系统经常说一些正确但没用的话，比如：

```text
投资有风险，请谨慎。
```

这句话没错，但对用户没有帮助。

后来我把它改成更具体的表达：

```text
如果跌破 MA20 且放量，原有观察逻辑降级。
如果仓位超过 50%，先处理集中度，而不是继续加仓。
如果市场是熊市环境，反弹先当修复，不当反转。
```

### 2. 不能假装百分百准确

股票分析不可能永远准确。

所以系统不是追求“永远说对”，而是追求：

- 判断有依据；
- 错了能知道为什么错；
- 有反证条件；
- 不编造缺失数据。

### 3. 免费公开数据有局限

公开接口能用，但不是交易级数据源。

所以系统要做多数据校验，也要明确告诉用户哪些数据缺失。

### 4. 界面要像工具，不像网页

界面也反复改了很多次。

普通网页风格不适合这个项目，因为用户打开它不是来看文章，而是要快速判断：

```text
今天账户有没有风险？
哪只股票最该处理？
市场现在能不能激进？
```

所以后面界面改得更像一个移动端操盘软件。

## 七、目前还可以继续改进的地方

现在这个系统还不是最终形态。

后续可以继续做：

- 更完整的行业轮动分析；
- 板块资金强弱排序；
- 龙虎榜和机构持仓变化；
- 财报关键指标拆解；
- 自定义仓位规则；
- 更准确的截图 OCR；
- 更专业的 K 线图交互；
- 多数据源互相校验。

如果以后接入更稳定的金融数据源，分析准确性还可以继续提升。

## 八、总结

这次项目从一个简单想法，逐步变成了一个比较完整的股票复盘工具。

它最重要的不是“预测明天涨跌”，而是帮助用户把信息整理成更清楚的判断：

```text
市场处于什么环境？
我的仓位是否过重？
这只股票有没有破线？
板块是否过度集中？
现在应该进攻、持有，还是防守？
```

我觉得这个项目最大的收获是：

> 一个 AI 工具如果只是把数据翻译成文字，其实价值有限。真正有用的是，它要能根据规则、证据和反证条件，帮用户形成更清楚的复盘框架。

这也是 Stock Lab 目前努力的方向。
